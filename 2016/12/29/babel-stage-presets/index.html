A quick reference on ECMAScript Maturity Stages

I wrote a quick reference on maturity Stages for ESnext Javascript features. It’s important be careful using features that are in stage 2 and below.

Babel has presets that will automatically include the Javascript features for that particular stage. I recommend you stick with preset-stage-3, as those features are the least likely to change, and are the most likely to be included in the yearly spec ratification.

To get an idea of which features are in which stage, reference the ECMAScript compatibility table.
